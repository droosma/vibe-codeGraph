# CodeGraph — Code Intelligence Graph

## What this is
A pre-built semantic graph of this codebase generated by Roslyn static analysis.
It maps every type, method, property and their relationships: calls, inheritance,
DI resolution, test coverage, and external dependencies.

## When to use this
BEFORE grepping or searching the codebase for a symbol, query the graph first.
A grep for a domain object like "Order" will return hundreds of results across
DTOs, controllers, services, tests, and migrations. The graph tells you the
actual dependency structure.

Use the graph when:
- Finding what calls a method or depends on a type
- Understanding IOC wiring (which concrete class resolves for an interface)
- Finding which tests cover a piece of code
- Tracing a call chain across projects
- Understanding the impact of changing a method signature

## How to query

```
codegraph query <symbol> [--depth N] [--kind TYPE] [--format context]
```

### Examples

```bash
# What calls OrderService.PlaceOrder?
codegraph query PlaceOrder --kind calls-to --depth 1

# Full dependency tree of OrderService, 2 levels deep
codegraph query OrderService --depth 2

# What tests cover the Reserve method?
codegraph query Reserve --kind covers

# What does IOrderRepository resolve to?
codegraph query IOrderRepository --kind resolves-to

# Everything related to Order in the Services project
codegraph query "*.Order*" --project MyApp.Services --depth 1
```

## Query options reference

| Option | Description | Default |
|--------|-------------|---------|
| `--depth N` | Traversal depth (0=node only) | 1 |
| `--kind TYPE` | Edge filter: calls-to, inherits, implements, depends-on, resolves-to, covers, all | all |
| `--namespace FILTER` | Namespace filter (supports wildcards) | none |
| `--project FILTER` | Project filter | none |
| `--format FMT` | Output: json, text, context | context |
| `--max-nodes N` | Cap output size | 50 |
| `--include-external` | Include external dependencies | false |
| `--graph-dir PATH` | Graph data directory | .codegraph |

## Staleness
The graph is indexed at a specific commit. If you are working on a feature branch,
the graph may not reflect recent changes. The query tool will warn you if the graph
is behind HEAD. To regenerate: `codegraph index --solution <path.sln> --output .codegraph/`

## Graph location
- Graph data: `.codegraph/graph/`
- Metadata: `.codegraph/meta.json`

## Workflow
1. User asks about a symbol or relationship → query the graph first
2. Use `--format context` output to identify relevant files and line ranges
3. Read ONLY those files/ranges for detailed context
4. This replaces broad grep/search for structural questions
