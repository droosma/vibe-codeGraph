# CodeGraph

A pre-built semantic graph of this codebase generated by Roslyn static analysis.
It maps every type, method, property and their relationships: calls, inheritance,
DI resolution, test coverage, and external dependencies.

## How to query

Run `codegraph query` via the bash tool. Prefer `--format context` — it returns
file paths and line ranges you can read directly.

```bash
codegraph query <symbol> [--depth N] [--kind TYPE] [--format context]
```

**Example 1 — call chain:**
Input: "What calls PlaceOrder?"
Output: `codegraph query PlaceOrder --kind calls-to --depth 1`

**Example 2 — DI resolution:**
Input: "What does IOrderRepository resolve to?"
Output: `codegraph query IOrderRepository --kind resolves-to`

**Example 3 — test coverage:**
Input: "What tests cover Reserve?"
Output: `codegraph query Reserve --kind covers`

**Example 4 — dependency tree:**
Input: "Show me the dependency tree of OrderService"
Output: `codegraph query OrderService --depth 2`

**Example 5 — scoped search:**
Input: "Everything related to Order in the Services project"
Output: `codegraph query "*.Order*" --project MyApp.Services --depth 1`

## Options

| Option | Description | Default |
|--------|-------------|---------|
| `--depth N` | Traversal depth (0 = node only) | 1 |
| `--kind TYPE` | calls-to, inherits, implements, depends-on, resolves-to, covers, all | all |
| `--namespace NS` | Namespace filter (wildcards OK) | none |
| `--project P` | Project filter | none |
| `--format FMT` | context, json, text | context |
| `--max-nodes N` | Cap output size | 50 |
| `--include-external` | Include external deps | false |
| `--graph-dir PATH` | Graph data directory | .codegraph |

## Workflow

1. Query the graph for the symbol or relationship
2. Use `--format context` output to get file paths and line ranges
3. Read ONLY those files/ranges — avoid broad grep/search
4. If the graph is stale or missing, fall back to grep and note reduced accuracy:
   > "The code graph is stale, so I'm using text search. Results may include
   > false positives from similarly-named symbols."

## Best practices

- Start with `--depth 1` and increase only if needed
- Use `--kind` filters to narrow results (e.g., `calls-to` instead of `all`)
- Combine `--project` and `--namespace` filters to reduce noise
- For interface resolution, always use `--kind resolves-to`
- For test coverage, always use `--kind covers`

## Graph info

- Data: `.codegraph/` (split by assembly)
- Metadata: `.codegraph/meta.json`
- Regenerate: `codegraph index --solution <path.sln> --output .codegraph/`
